-- При помощи базы данных Redis решите задачу поиска имени пользователя по электронному адресу и наоборот, поиск электронного адреса пользователя по его имени.

Ответ:
Если имя пользователя и электронный адрес уникальны оба, тогда можно просто хранить в БД пару записей, у первой из которых ключ - имя, значение - адрес, а у второй - наоборот, ключ - адрес, значение - имя пользователя.
Если уникален электронный адрес, а имя - нет, тогда нужно хранить запись, у которой ключ - адрес, значение - имя. Для поиска электронного адреса по имени нужно создать множество (set), имеющее имя, равное имени пользователя, и содержащее все адреса, которые имеют пользователи с одинаковым именем. Этот способ не очень хорош, на мой взгляд, т.к. за сохранение быстродействия, которым так полезен Redis, мы расплачиваемся минимум многократным перерасходом памяти.
Ситуация, когда электронный адрес не уникален, предлагаю не рассматривать, т.к. в моем понимании хорошо спроектированная система не должна допускать такой ситуации.

Пример из командной строки:
127.0.0.1:6379> SET veronika v.roznyuk@gmail.com
OK
127.0.0.1:6379> SET v.roznyuk@gmail.com veronika
OK
127.0.0.1:6379> GET veronika
"v.roznyuk@gmail.com"
127.0.0.1:6379> GET v.roznyuk@gmail.com
"veronika"
...
127.0.0.1:6379> SET v.roznyuk@gmail.com veronika
OK
127.0.0.1:6379> SET v.stepanova@mail.ru victoria
OK
127.0.0.1:6379> SET roznyuk_vm@yandex.ru veronika
OK
127.0.0.1:6379> DEL veronika
(integer) 1
127.0.0.1:6379> SADD veronika v.roznyuk@gmail.com roznyuk_vm@yandex.ru
(integer) 2
127.0.0.1:6379> SADD victoria v.stepanova@mail.ru
(integer) 1
127.0.0.1:6379> SMEMBERS veronika
1) "roznyuk_vm@yandex.ru"
2) "v.roznyuk@gmail.com"
127.0.0.1:6379> SMEMBERS victoria
1) "v.stepanova@mail.ru"
127.0.0.1:6379>
